# AGENTS.md — LFX 辭海網站專案指示

## 必讀資料

在分析、修改、撰寫或重構本專案之前，先閱讀：

- `LFX_PERSONAL_CONTEXT.md`
- `CONTENT_PLAN.md`
- 如果本機存在 `LFX_PRIVATE_CONTEXT.md`，再讀取該私人完整版。

`LFX_PERSONAL_CONTEXT.md` 是可留在公開 repository 的安全基準；`CONTENT_PLAN.md` 是首頁資訊層級、內容展示與互動方式的長期設計基準；`LFX_PRIVATE_CONTEXT.md` 是只保留在本機的完整私人背景資料。

## 公開與私人資料分離

- `LFX_PRIVATE_CONTEXT.md` 不得被 Git 追蹤、commit 或 push。
- 私人完整版只供理解本人背景，不代表其中內容已授權公開。
- 不得把私人完整版的內容直接寫入網站、`data/`、元件、README 或其他公開文件。
- 只有本人明確授權公開的具體內容，才能從私人完整版整理到公開網站資料。
- 若公開版與私人版的使用邊界不明確，先詢問本人，不自行判定為可公開。

## 身分隔離

- 本專案只服務「LFX 辭海」本人。
- 不得使用專案外可能屬於其弟弟的機器人、ROS 2、STM32、嵌入式、PCB、強化學習等資料。
- 本人是 BIM／Revit／Dynamo 工程使用者，不是機器人或嵌入式工程師。

## 修改原則

- 先介紹人，再介紹工作、興趣、設備與作品。
- 保留現有 Next.js、TypeScript、Tailwind 架構。
- 不重建專案，不擅自增加多頁、資料庫、後台或文章系統。
- 將內容集中於 `data/`，避免重複寫死。
- 不虛構學歷、公司、年資、車輛、規格、作品成果、社群帳號或人格故事。
- 資訊不確定時，先提問再修改。
- 個性與展望目前只搭骨架，不自行定稿。
- 私密人格與生活內容不得未經確認公開。
- 修改後執行 `npm run build` 並回報結果。

## 個人品牌與首頁資訊層級

- 本站是個人品牌網站，不是部落格、履歷表或工程公司官網。
- 首頁第一層以視覺、真實照片、大字、短句、標籤與留白為主。
- 訪客應能快速知道 LFX 是誰、做什麼、重視什麼，並感受到本人魅力與辨識度。
- 完整故事與細節必須保留，但放入第二層展開內容，不得全部直接攤在首頁。
- About、Journey、Life 一律規劃成「第一層精準摘要＋第二層完整故事」的兩層資訊結構。
- 不得把所有第二層內容統一做成 FAQ 式 Accordion。
- 桌面版優先評估右側 Drawer 或視覺化展開面板；手機版可使用卡片內展開或全螢幕閱讀。
- Life 應提高真實照片與影片的比例，重機、電腦、影片及日常以視覺內容為主。
- 資訊完整不能以犧牲視覺張力、留白、質感與個人辨識度為代價。
- 具體規劃、文案限制與驗收標準以 `CONTENT_PLAN.md` 為準。
